## 3.1 Powershell放宽当前用户的脚本策略

## 3.2 启动GateWay

================================================================================================

OpenClaw 如何运行起来 — 常见方式说明（Windows 环境为主）

生成依据：OpenClaw CLI 2026.3.13 的 `openclaw gateway --help`、`openclaw daemon --help`、 `openclaw tui --help`、`openclaw dashboard --help`

官方文档索引：Gateway CLI：<https://docs.openclaw.ai/cli/gateway> 总 CLI 索引：<https://docs.openclaw.ai/cli>

================================================================================================

GateWay核心概念（先理解再启动）

1. Gateway（网关）是本地 WebSocket 服务，多数功能（TUI、Dashboard、agent 调用等） 都依赖「网关已启动且可连接」。

2. 默认配置里网关地址一般为：<ws://127.0.0.1:18789（具体以> openclaw.json 里 gateway.port 为准）。

3. 若配置了 gateway.auth.mode=token，客户端连接时需要携带与配置一致的 token （或通过环境变量 OPENCLAW\_GATEWAY\_TOKEN 等，见 CLI 帮助）。

================================================================================================

# 四、使用

## 4.0 注意事项

数据安全：部署openclaw的电脑数据应有备份或合理的git仓库管理，网上存在openclaw误删数据文件情况

代码安全：公司内部

## 4.1 HTT测试工具(自动化执行)

可通过沟通对话方式教会openclaw实现定时执行HTT测试

================================================================================================

OpenClaw 如何运行起来 — 常见方式说明（Windows 环境为主）

生成依据：OpenClaw CLI 2026.3.13 的 `openclaw gateway --help`、`openclaw daemon --help`、 `openclaw tui --help`、`openclaw dashboard --help`

官方文档索引：Gateway CLI：<https://docs.openclaw.ai/cli/gateway> 总 CLI 索引：<https://docs.openclaw.ai/cli>

================================================================================================

GateWay核心概念（先理解再启动）

1. Gateway（网关）是本地 WebSocket 服务，多数功能（TUI、Dashboard、agent 调用等） 都依赖「网关已启动且可连接」。

2. 默认配置里网关地址一般为：<ws://127.0.0.1:18789（具体以> openclaw.json 里 gateway.port 为准）。

3. 若配置了 gateway.auth.mode=token，客户端连接时需要携带与配置一致的 token （或通过环境变量 OPENCLAW\_GATEWAY\_TOKEN 等，见 CLI 帮助）。
