# [OpenClaw-Termux 安卓安装与使用教程](#openclaw-termux-安卓安装与使用教程)

本文基于 OpenClaw 官方仓库整理，适用于你当前目录中的 `OpenClaw-v1.8.7-universal.apk` 安装包。

* 官方仓库: <https://github.com/mithun50/openclaw-termux>
* 当前 APK 文件: `OpenClaw-v1.8.7-universal.apk`
* 当前 APK SHA-256: `A01AEFFE741788CDDE5A367A91FDF594CD4E88F8DB4326B47D8E456AC9204F6F`

## [一、软件简介](#一软件简介)

OpenClaw-Termux 是一个可在 Android 上运行的 OpenClaw AI Gateway 图形化应用。根据官方说明，它会在 Android 设备中自动准备 Ubuntu proot 环境、Node.js 22 和 OpenClaw，并提供内置终端、Dashboard、日志查看、设置管理等功能。

官方建议环境如下：

* Android 10 及以上
* 预留大约 500MB 以上存储空间
* 首次配置时保持网络稳定
* 为避免后台被系统杀掉，建议关闭该应用的电池优化

## [三、ADB 连接设备并安装 APK](#三adb-连接设备并安装-apk)

本节演示通过 ADB 将 `OpenClaw-v1.8.7-universal.apk` 安装到 Android 设备。

### [1. 准备工作](#1-准备工作)

请先确认电脑已经安装并配置好系统 `adb`。

可先执行：

```
adb version
```

如果终端提示找不到 `adb` 命令，说明当前电脑还没有安装 Android Platform Tools，请先安装 ADB 并确认 `adb` 已加入系统环境变量，然后再继续后续步骤。

在 Android 手机或平板上完成以下操作：

1. 打开“设置”。
2. 连续点击“版本号”或“Build Number”数次，开启开发者选项。
3. 进入“开发者选项”，开启“USB 调试”。
4. 使用 USB 数据线把设备连接到电脑。
5. 手机上若弹出“是否允许 USB 调试”，点击“允许”。

### [2. 检查系统 ADB 是否可用](#2-检查系统-adb-是否可用)

建议在 PowerShell 或命令提示符中直接使用系统 `adb` 命令，无需写具体路径。

### [3. 检查设备连接状态](#3-检查设备连接状态)

```
adb devices
```

正常情况下会看到类似输出：

```
List of devices attached
XXXXXXXX device
```

如果看到 `unauthorized`，请查看手机屏幕并确认调试授权。

### [4. 安装 OpenClaw APK](#4-安装-openclaw-apk)

```
adb install -r .\OpenClaw-v1.8.7-universal.apk
```

参数说明：

* `install`：安装 APK
* `-r`：如果设备中已安装旧版本，则执行覆盖安装并尽量保留应用数据

如果设备里已经装过签名不同的版本，可能会报安装失败。此时建议先在手机上手动卸载旧版本，然后重新执行安装命令：

```
adb install .\OpenClaw-v1.8.7-universal.apk
```

### [5. 可选：无线 ADB 安装](#5-可选无线-adb-安装)

如果你已经能通过 USB 正常识别设备，也可以切换为无线 ADB：

```
adb tcpip 5555
adb connect 手机IP:5555
adb install -r .\OpenClaw-v1.8.7-universal.apk
```

## [四、安装完成后的首次配置与运行教程](#四安装完成后的首次配置与运行教程)

根据官方说明，OpenClaw 首次运行时会自动准备 Ubuntu rootfs、Node.js 22 和 OpenClaw 环境，所以第一次配置耗时会明显长一些。

### [1. 首次打开应用](#1-首次打开应用)

安装完成后，在手机中打开 OpenClaw。

首次打开时建议注意以下事项：

* 保持网络畅通，因为首次配置会下载较多内容
* 确保设备剩余空间充足
* 不要随意授予“全部文件访问权限”

官方特别提醒：

* 这个应用正常运行并不需要默认授予完整存储权限
* 如果只是运行 OpenClaw，遇到存储授权提示时，一般应选择拒绝
* 只有你明确需要让 proot 访问 `/sdcard` 时，才考虑授权

### [2. 点击 Begin Setup 开始环境部署](#2-点击-begin-setup-开始环境部署)

进入应用后，点击 `Begin Setup`。

此过程会自动完成以下内容：

* 下载并部署 Ubuntu proot 环境
* 安装 Node.js 22
* 安装 OpenClaw
* 准备应用运行所需的基础依赖

首次部署时，建议：

* 保持应用在前台
* 不要中途切出太久
* 不要让系统清理后台任务

### [3. 可选安装开发工具](#3-可选安装开发工具)

官方说明中提到，基础环境完成后，还可以从应用里按需安装额外工具：

* Go
* Homebrew
* OpenSSH

这些工具不是启动 OpenClaw 的必须项。如果只是使用 OpenClaw 网关与 Dashboard，前期可以先不装，等后续需要时再安装。

### [4. 完成 Onboarding 初始配置](#4-完成-onboarding-初始配置)

环境准备完成后，进入 `Onboarding` 配置流程。

这里主要做几件事：

* 选择绑定方式
* 配置 AI Provider 的 API Key
* 生成并保存 Dashboard 的访问 Token URL

根据官方说明，非 root 设备建议选择：

```
Loopback (127.0.0.1)
```

也就是说，绑定地址优先选本机回环地址 `127.0.0.1`。

### [5. 配置 API Key](#5-配置-api-key)

在 Onboarding 过程中，按你的实际需要填写对应服务商的 API Key。官方 README 中提到应用支持多个 Provider，例如：

* OpenAI
* Anthropic
* Google Gemini
* OpenRouter
* NVIDIA NIM
* DeepSeek
* xAI

如果你只使用其中一种，先配置一个即可，后续可在应用中继续增删或修改。

### [6. 启动 OpenClaw Gateway](#6-启动-openclaw-gateway)

配置完成后，进入 Dashboard 页面，点击：

```
Start Gateway
```

启动成功后，一般可以看到：

* 网关运行状态
* 认证 Token URL
* 相关控制按钮

官方说明中提到，应用会自动捕获类似下面这种地址：

```
http://localhost:18789/#token=...
```

这个地址可用于打开 Web Dashboard。

### [7. 关闭电池优化](#7-关闭电池优化)

这是官方特别强调的一步。

如果不关闭电池优化，Android 可能会在后台把 OpenClaw 网关杀掉，导致：

* 网关运行一段时间后自动停止
* Dashboard 无法打开
* 日志中出现中断

建议路径通常为：

```
系统设置 > 应用 > OpenClaw > 电池 > 不受限制
```

不同品牌手机界面略有不同，但核心目标是让 OpenClaw 不被系统省电策略限制。

## [五、软件内使用教程](#五软件内使用教程)

安装和初始化完成后，日常使用主要集中在以下几个页面。

### [1. Dashboard 主页面](#1-dashboard-主页面)

Dashboard 是日常使用的主入口，通常可以完成以下操作：

* 启动或停止 Gateway
* 查看当前运行状态
* 查看 Token URL
* 打开 Web Dashboard
* 快速进入日志、终端、设置、包管理等功能

推荐日常使用流程：

1. 打开 OpenClaw。
2. 进入 Dashboard。
3. 点击 `Start Gateway`。
4. 确认状态正常。
5. 通过内置 WebView 或 Token URL 进入 Web Dashboard。

### [2. Web Dashboard](#2-web-dashboard)

官方说明中，OpenClaw 的网页控制台地址通常是：

```
http://localhost:18789/#token=...
```

应用会把这个地址自动带上认证 Token，并在内置 WebView 中打开。

在 Web Dashboard 中，你可以做的事情通常包括：

* 查看当前会话状态
* 发送控制命令
* 管理当前交互会话

官方 README 中列出的常用命令示例有：

```
/status
/think high
/reset
```

可理解为：

* `/status`：查看当前网关状态
* `/think high`：开启高质量思考模式
* `/reset`：重置当前会话

### [3. Onboarding / Provider 配置](#3-onboarding--provider-配置)

如果后续需要调整模型供应商或 API Key，可再次进入相关配置页面。

常见用途：

* 更换 API Key
* 新增其他 Provider
* 切换使用模型
* 重新生成或确认访问配置

如果某个 Provider 无法使用，优先检查：

* API Key 是否正确
* 网络是否正常
* 当前选择的模型是否可用

### [4. Terminal 内置终端](#4-terminal-内置终端)

OpenClaw 自带内置终端，可直接用于查看或执行底层环境命令。

它的作用包括：

* 查看 Ubuntu / proot 环境
* 手动执行 OpenClaw 相关命令
* 排查安装异常
* 进行更高级的运维操作

如果需要在终端里手动检查状态，可尝试执行官方 README 中提到的常用命令：

```
openclawx status
openclawx start
openclawx onboarding
openclawx shell
openclawx doctor
openclawx gateway --verbose
```

这些命令分别用于：

* `openclawx status`：查看安装与运行状态
* `openclawx start`：启动网关
* `openclawx onboarding`：重新执行向导配置
* `openclawx shell`：进入 Ubuntu shell
* `openclawx doctor`：做环境检查
* `openclawx gateway --verbose`：以详细日志方式启动网关

### [5. Logs 日志页面](#5-logs-日志页面)

如果网关启动失败、页面打不开或 Provider 调用异常，优先查看 `Logs` 页面。

日志页面适合排查以下问题：

* 首次安装是否完整
* Node.js / OpenClaw 是否正常启动
* Gateway 是否异常退出
* Token URL 是否生成
* Provider 配置是否报错

推荐排错顺序：

1. 先看 Dashboard 是否显示正在运行。
2. 若未运行，进入 Logs 查看最近报错。
3. 必要时重新执行 Onboarding。
4. 仍异常时，回到 Setup 或终端执行诊断命令。

### [6. Settings 设置页面](#6-settings-设置页面)

设置页面通常可用于：

* 查看系统与安装状态
* 检查可选包安装情况
* 重新运行 Setup
* 处理电池优化相关配置
* 管理部分权限与偏好选项

建议重点关注：

* 电池优化是否关闭
* 基础环境是否已完整安装
* 是否误开启了不必要的存储权限

### [7. SSH 与可选功能](#7-ssh-与可选功能)

根据官方文档，应用还支持 SSH 远程访问及一些设备能力扩展。普通用户前期可以先不启用，等熟悉基础使用后再逐步尝试。

更适合进阶用户的功能包括：

* SSH Server 管理
* 设备能力接入
* 可选开发工具安装

## [六、常见问题与处理建议](#六常见问题与处理建议)

### [1. 首次配置很慢](#1-首次配置很慢)

这是正常现象。因为首次配置会下载并安装 Ubuntu rootfs、Node.js 和 OpenClaw，耗时受网络速度和设备性能影响较大。

### [2. Gateway 无法启动](#2-gateway-无法启动)

可以按以下顺序排查：

1. 确认首次 Setup 已完整执行完毕。
2. 确认 Onboarding 已完成。
3. 检查 API Key 是否填写正确。
4. 查看 Logs 页面。
5. 关闭电池优化后再试一次。

### [3. RK3576 开发板运行结果](#3-rk3576-开发板运行结果)

在 RK3576 开发板上的测试结果如下：

* 应用终端可以正常进入
* Web Dashboard 可以正常打开
* 但实际执行操作时整体比较卡顿

这说明 OpenClaw 在 RK3576 开发板环境中基本可以跑起来，但当前体验偏慢，不适合作为高流畅度演示环境。若要继续使用，建议优先关注以下几点：

* 首次安装和依赖下载阶段尽量保持网络稳定
* 确保开发板剩余存储空间充足
* 尽量减少后台程序占用
* 如需更流畅的体验，优先在性能更强的 Android 设备上运行

### [4. 程序后台运行一段时间后自动停止](#4-程序后台运行一段时间后自动停止)

大概率是 Android 电池优化导致。请把 OpenClaw 设置为“不受限制”或同等含义的后台运行模式。

### [5. 是否一定要授予存储权限](#5-是否一定要授予存储权限)

不一定。官方已明确说明，应用默认不需要完整存储权限即可运行。除非你确实需要让 proot 访问 `/sdcard`，否则建议不要开启。

### [6. Web Dashboard 打不开](#6-web-dashboard-打不开)

优先检查：

* Gateway 是否已启动
* Token URL 是否已生成
* 当前是否仍在本机回环地址模式
* 应用是否被系统杀后台

## [七、QQ / 飞书机器人接入与远程指挥说明](#七qq--飞书机器人接入与远程指挥说明)

这一节讲的是把 QQ 或飞书机器人接入到当前这台已经部署 OpenClaw 的设备上。接入完成后，你就可以在另一台设备上的 QQ 或飞书聊天界面中，直接给这个 OpenClaw 实例发消息、下指令和查看回复。

可理解为：

* OpenClaw 部署在当前安卓设备或开发板上
* QQ / 飞书 作为消息入口
* 你在另一台手机、平板或电脑上打开 QQ / 飞书
* 通过聊天窗口与机器人对话，间接指挥这台运行 OpenClaw 的设备

### [1. 实现原理](#1-实现原理)

整体链路如下：

```
另一台设备上的 QQ / 飞书客户端
↓
QQ 机器人 / 飞书机器人
↓
当前设备上的 OpenClaw Gateway
↓
OpenClaw Agent / 已配置模型 / 本机能力
```

也就是说，后续你不一定非要在 OpenClaw 本机界面里操作。只要机器人通道接好，并且 Gateway 正常运行，就可以在别的设备上的聊天窗口里远程发起交互。

### [2. 使用前提](#2-使用前提)

开始前请先确认：

* OpenClaw 已在当前设备安装完成
* `Begin Setup`、`Onboarding`、`Start Gateway` 已经跑通
* 已配置至少一个可用的模型 Provider 和 API Key
* 当前设备网络正常，且 OpenClaw 没有被系统杀后台

建议优先在 OpenClaw 的 `Terminal` 或 `Logs` 页面中确认网关处于可用状态，再继续接入 QQ 或飞书通道。

### [3. 飞书机器人接入 OpenClaw](#3-飞书机器人接入-openclaw)

飞书是 OpenClaw 官方文档中已有的消息通道之一。根据官方文档，快速接入方式是先在终端执行：

```
openclaw channels login --channel feishu
```

执行后，终端会显示二维码。使用你的飞书手机客户端扫描该二维码，即可创建并绑定飞书机器人。完成后重启网关：

```
openclaw gateway restart
```

接入成功后，你可以在另一台设备上：

* 打开飞书客户端
* 找到对应机器人
* 通过私聊或群聊 `@机器人` 的方式发送消息

常见用法：

* 私聊机器人，直接发自然语言指令
* 群聊里 `@机器人` 后发送命令
* 发送 `/status`、`/reset`、`/model` 等文本命令

根据官方文档，飞书通道默认有这些特点：

* 私聊可通过 `dmPolicy` 控制谁能使用
* 群聊默认通常要求 `@机器人` 才会响应
* 群聊默认更偏向白名单控制

如果你希望在飞书群里更方便使用，通常需要进一步配置：

* 是否允许所有群
* 是否必须 `@机器人`
* 哪些用户或群能访问

### [4. 飞书通道的远程控制方式](#4-飞书通道的远程控制方式)

飞书接入成功后，远程控制的典型方式如下：

1. 在部署了 OpenClaw 的设备上保持 Gateway 运行。
2. 在另一台设备上打开飞书。
3. 给机器人发送任务，例如“查看当前状态”“执行某项部署检查”“读取日志摘要”等。
4. 机器人把消息转给当前设备上的 OpenClaw。
5. OpenClaw 返回结果，再由飞书机器人回复到聊天界面。

这意味着你可以不直接接触部署设备本身，而是在另一台常用设备上通过飞书完成大部分交互。

### [5. QQ 机器人接入 OpenClaw](#5-qq-机器人接入-openclaw)

QQ 机器人通常通过 QQ Bot 插件接入 OpenClaw。该方案不是 `openclaw-termux` 仓库内置说明的一部分，而是基于 OpenClaw 生态中的 QQ Bot 插件实现。

整体步骤通常如下：

1. 先在 QQ 开放平台创建 QQ Bot。
2. 获取机器人的 `AppID` 和 `AppSecret`。
3. 在 OpenClaw 所在设备的终端中安装 QQ Bot 插件。
4. 把 `AppID` 和 `AppSecret` 配到 OpenClaw。
5. 重启 Gateway。

根据插件文档，手动安装命令通常为：

```
openclaw plugins install @tencent-connect/openclaw-qqbot@latest
openclaw channels add --channel qqbot --token "AppID:AppSecret"
openclaw gateway restart
```

完成后，你可以在另一台设备上的 QQ 中：

* 打开该机器人会话
* 直接私聊机器人
* 或在群里 `@机器人`

然后通过聊天界面把任务发送给当前这台运行 OpenClaw 的设备。

### [6. QQ 通道的远程控制方式](#6-qq-通道的远程控制方式)

QQ 通道接通后，使用逻辑和飞书类似：

1. 当前设备运行 OpenClaw Gateway。
2. 另一台设备打开 QQ。
3. 在 QQ 聊天窗口里给机器人发消息。
4. 机器人把消息转发给当前设备上的 OpenClaw。
5. OpenClaw 执行并返回结果。

QQ Bot 插件还提供了一些插件级命令，常见示例包括：

* `/bot-ping`
* `/bot-version`
* `/bot-help`
* `/bot-logs`

这些命令主要用于插件状态检查、版本确认和日志导出。

### [7. 适合远程下发的操作](#7-适合远程下发的操作)

当 QQ 或飞书机器人接好以后，比较适合在聊天窗口中做这些事情：

* 查询 OpenClaw 当前状态
* 发起普通对话和任务指令
* 检查部署状态或运行日志
* 让 Agent 帮你整理信息、分析问题、输出命令建议

如果你已经把某些设备能力、节点能力或插件能力接入 OpenClaw，也可以继续通过聊天窗口间接调用这些能力。

### [8. 使用限制与注意事项](#8-使用限制与注意事项)

虽然接入 QQ / 飞书后可以在别的设备上远程聊天，但仍需注意下面几点：

* 聊天窗口只是“消息入口”，真正执行任务的还是当前这台部署 OpenClaw 的设备
* 如果当前设备上的 Gateway 停止了，机器人就不会正常响应
* 如果当前设备被系统省电策略限制，远程消息可能会收不到或回复很慢
* 飞书和 QQ 的权限控制、群聊响应规则、审核要求并不完全一样
* 机器人能否执行某些高级动作，还取决于当前 OpenClaw 的模型、权限、插件和本机环境

### [9. 安卓设备上的建议做法](#9-安卓设备上的建议做法)

如果你准备在安卓设备或 RK3576 开发板上长期使用 QQ / 飞书远程控制，建议：

* 关闭 OpenClaw 的电池优化
* 尽量保持应用常驻前台或使用稳定的后台保活方案
* 先在本机 Dashboard 中验证 Gateway 稳定运行
* 再配置飞书或 QQ 通道
* 接入完成后，优先从另一台设备做简单消息测试，再逐步尝试更复杂的任务

### [10. 说明](#10-说明)

本教程中的飞书接入说明基于 OpenClaw 官方 Feishu / Lark 文档整理；QQ 接入说明基于 OpenClaw 生态中的 QQ Bot 插件文档整理。由于 QQ 方案依赖额外插件，后续如果插件安装命令、参数名或平台要求发生变化，应以插件仓库最新文档为准。

## [八、推荐使用流程](#八推荐使用流程)

如果你是第一次使用，建议按下面的顺序操作：

1. 用 ADB 安装 `OpenClaw-v1.8.7-universal.apk`。
2. 打开应用，点击 `Begin Setup`。
3. 等待 Ubuntu、Node.js、OpenClaw 自动安装完成。
4. 进入 `Onboarding`。
5. 绑定方式选择 `Loopback (127.0.0.1)`。
6. 配置你要使用的 API Key。
7. 回到 Dashboard，点击 `Start Gateway`。
8. 打开内置 Web Dashboard 开始使用。
9. 到系统里关闭 OpenClaw 的电池优化。
