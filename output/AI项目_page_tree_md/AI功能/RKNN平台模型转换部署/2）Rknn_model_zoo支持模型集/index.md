# [`rknn_model_zoo` 三份文档阅读说明](#rknn-model-zoo-三份文档阅读说明)

本文说明以下三份 Markdown **各自用途、内容结构、建议阅读顺序**，并与仓库内真实路径、章节标题对齐：

* `rknn_model_zoo/docs/Compilation_Environment_Setup_Guide_CN.md` — 交叉编译环境搭建指南（中文）
* `rknn_model_zoo/examples/whisper/README_中文.md` — Whisper 示例说明（中文）
* `rknn_model_zoo/examples/mms_tts/README_中文.md` — MMS-TTS 示例说明（中文）

---

## [1. 三份文档一句话对照](#1-三份文档一句话对照)

| 文档 | 定位 | 主要解决什么问题 |
| --- | --- | --- |
| **Compilation\_Environment\_Setup\_Guide\_CN.md** | **全仓库通用**的交叉编译前置条件 | 在 x86 Linux 上如何准备 **NDK / GCC**，才能用根目录 `build-android.sh`、`build-linux.sh` 编译**任意**示例的 **C/C++ Demo** |
| **examples/whisper/README\_中文.md** | **Whisper 语音识别**示例专篇 | 该示例的模型下载、`convert.py` 转 RKNN、Python Demo、Android/Linux C Demo 命令与参数（`en`/`zh` 任务等） |
| **examples/mms\_tts/README\_中文.md** | **MMS-TTS 语音合成**示例专篇 | 该示例的模型下载、转 RKNN、Python/Android/Linux 流程及 `speaking_rate`、`output.wav` 等用法 |

---

## [2. 各文档在源码里「声明」的职责](#2-各文档在源码里声明的职责)

### [2.1 交叉编译环境搭建指南（`docs/Compilation_Environment_Setup_Guide_CN.md`）](#21-交叉编译环境搭建指南docscompilation-environment-setup-guide-cnmd)

开篇即限定范围：只有搭好交叉编译环境，才能在 x86 Linux 上编译本工程示例中的 **C/C++ Demo**。

```
# 交叉编译环境搭建指南

需要搭建好交叉编译环境，才可以在x86 Linux系统上编译本工程示例中的C/C++ Demo。
```

**结构要点**：

* **Android**：通过环境变量 `ANDROID_NDK_PATH` 指向 NDK，再调用 `./build-android.sh -t … -a … -d <model_name>`；文档给出 NDK 下载与参数说明（见同文件第 6–41 行附近）。
* **Linux**：通过 `GCC_COMPILER` 指向交叉编译器前缀路径，再调用 `./build-linux.sh`；文档按 `aarch64` / `armhf` / `armhf-uclibcgnueabihf(RV1103/RV1106)` 给出工具链下载与 `GCC_COMPILER` 示例（见同文件第 43–100 行附近）。

**它不负责**：某个具体模型（如 Whisper）的 ONNX 下载地址、Python 推理参数等——这些在各 `examples/<模型>/README` 中。

---

### [2.2 Whisper 中文 README（`examples/whisper/README_中文.md`）](#22-whisper-中文-readmeexampleswhisperreadme-中文md)

**第 1 节**说明示例背景：Whisper 为通用语音识别模型，并给出上游仓库链接。

```
## 1. 简介

Whisper 是一个通用语音识别模型。它在大规模、多样化音频数据集上训练，同时也是一个多任务模型，可执行多语言语音识别、语音翻译和语言识别任务。

本示例使用的模型来自以下开源项目：

https://github.com/openai/whisper
```

**目录骨架**（与文件内「目录」锚点一致）：简介 → 支持平台 → 预训练模型与下载 → **转换为 RKNN** → **Python Demo** → **Android Demo**（编译 / 推送 / 运行）→ **Linux Demo**（同上）→ 预期结果。 其中 **Linux Demo** 明确指向全仓库编译说明：`../../docs/Compilation_Environment_Setup_Guide.md`（英文文件名；与中文版 `Compilation_Environment_Setup_Guide_CN.md` 为同一类文档）。

```
## 7. Linux Demo

请注意，Linux 编译工具链推荐使用 `gcc-linaro-6.3.1(aarch64)/gcc-arm-8.3(armhf)/armhf-uclibcgnueabihf(armhf for RV1106/RV1103 series)`。使用其他版本可能会遇到 C demo 编译失败问题。详细编译说明请参考 [Compilation_Environment_Setup_Guide.md](../../docs/Compilation_Environment_Setup_Guide.md)。
```

### [2.3 MMS-TTS 中文 README（`examples/mms_tts/README_中文.md`）](#23-mms-tts-中文-readmeexamplesmms-ttsreadme-中文md)

**第 1 节**说明 MMS-TTS 来源与 Hugging Face 模型页。

```
## 1. 说明

MMS-TTS（Massively Multilingual Speech, Text to Speech）是 Facebook 大规模多语言语音项目的一部分，目标是在多种语言场景下提供语音技术能力。

本示例使用的模型来自以下开源项目：

https://huggingface.co/facebook/mms-tts-eng
```

**章节结构与 Whisper 中文 README 平行**：说明 → 平台 → 模型下载 → `convert.py` → Python Demo → Android → Linux → 预期结果；差异在 **业务**：TTS 输入为文本，板端生成 `output.wav`，并文档化 `speaking_rate` 可选参数（见该 README 第 128–145、190–207 行附近）。 **Linux Demo** 同样引用 `../../docs/Compilation_Environment_Setup_Guide.md` 作为交叉编译细节入口（第 148–150 行附近）。

## [3. 建议怎么读：顺序与分叉](#3-建议怎么读顺序与分叉)

### [3.1 总体原则](#31-总体原则)

```
全仓库要编 C/C++ Demo（Android 或 Linux）
→ 先读 docs/Compilation_Environment_Setup_Guide_CN.md
→ 再进入具体 examples/<模型>/README_中文.md 按第 4～7 节操作

只做 Python（ONNX 或 PC 连板 RKNN）不编板端 C Demo
→ 可跳过交叉编译指南，直接从对应 README 第 3～5 节开始
```

### [3.2 Whisper 与 MMS-TTS 两份 README 的关系](#32-whisper-与-mms-tts-两份-readme-的关系)

二者**不是**「先读 A 再读 B」的依赖关系，而是**两条并列示例线**：

| 若你的目标 | 以哪份 README 为主 |
| --- | --- |
| 语音识别（ASR）、Whisper 双模型（encoder/decoder）、`--task en/zh` | `examples/whisper/README_中文.md` |
| 英语 TTS、文本→wav、`speaking_rate` | `examples/mms_tts/README_中文.md` |

当你要在 **Linux 板**上编译其中**任一**示例的 C Demo 时，两份 README 都会把你指回 **`docs/Compilation_Environment_Setup_Guide*.md`** 查 `GCC_COMPILER` 与架构对应关系；此时**交叉编译指南**是共同前置。

### [3.3 与中英文编译指南的对应](#33-与中英文编译指南的对应)

Whisper / MMS-TTS 的 Linux 章节内链为 **`Compilation_Environment_Setup_Guide.md`**（英文）。若你更习惯中文，可直接阅读 **`Compilation_Environment_Setup_Guide_CN.md`**，变量名与脚本用法与英文版一致，仅语言不同。
