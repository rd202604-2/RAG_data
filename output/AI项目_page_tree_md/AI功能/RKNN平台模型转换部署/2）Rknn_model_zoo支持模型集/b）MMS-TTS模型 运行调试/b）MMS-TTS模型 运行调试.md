## [1. 说明](#1-说明)

MMS-TTS（Massively Multilingual Speech, Text to Speech）是 Facebook 大规模多语言语音项目的一部分，目标是在多种语言场景下提供语音技术能力。

本示例使用的模型来自以下开源项目：

<https://huggingface.co/facebook/mms-tts-eng>

## [2. 当前支持平台](#2-当前支持平台)

RK3562, RK3566, RK3568, RK3576, RK3588, RV1126B

## [3. 预训练模型](#3-预训练模型)

下载链接：

[mms\_tts\_eng\_encoder\_200.onnx](https://ftrg.zbox.filez.com/v2/delivery/data/95f00b0fc900458ba134f8b180b3f7a1/examples/mms_tts/mms_tts_eng_encoder_200.onnx)<br />[mms\_tts\_eng\_decoder\_200.onnx](https://ftrg.zbox.filez.com/v2/delivery/data/95f00b0fc900458ba134f8b180b3f7a1/examples/mms_tts/mms_tts_eng_decoder_200.onnx)

使用 shell 命令下载：

```
cd model
./download_model.sh
```

**注意：如需导出 mms\_tts ONNX 模型，请参考 [export\_onnx.md](./export_onnx.md)**

## [4. 转换为 RKNN](#4-转换为-rknn)

*用法：*

```
cd python
python convert.py <onnx_model> <TARGET_PLATFORM> <dtype(optional)> <output_rknn_path(optional)>

# 例如： 
python convert.py ../model/mms_tts_eng_encoder_200.onnx rk3588
# 输出模型将保存为 ../model/mms_tts_eng_encoder_200.rknn

python convert.py ../model/mms_tts_eng_decoder_200.onnx rk3588
# 输出模型将保存为 ../model/mms_tts_eng_decoder_200.rknn
```

*说明：*

* `<onnx_model>`：指定 ONNX 模型路径。
* `<TARGET_PLATFORM>`：指定 NPU 平台名。支持平台见 [这里](#2-%E5%BD%93%E5%89%8D%E6%94%AF%E6%8C%81%E5%B9%B3%E5%8F%B0)。
* `<dtype>(optional)`：可选 `i8` 或 `fp`。`i8` 表示量化，`fp` 表示不量化。默认值为 `fp`。
* `<output_rknn_path>(optional)`：指定 RKNN 模型保存路径，默认保存到 ONNX 模型同目录。

## [5. Python Demo](#5-python-demo)

*用法：*

```
cd python
# 使用 ONNX 模型推理
python mms_tts.py --encoder_model_path <onnx_model> --decoder_model_path <onnx_model>

# 使用 RKNN 模型推理
python mms_tts.py --encoder_model_path <rknn_model> --decoder_model_path <rknn_model> --target <TARGET_PLATFORM>
```

*说明：*

* `<TARGET_PLATFORM>`：指定 NPU 平台名。支持平台见 [这里](#2-%E5%BD%93%E5%89%8D%E6%94%AF%E6%8C%81%E5%B9%B3%E5%8F%B0)。
* `<onnx_model / rknn_model>`：指定模型路径。

## [6. Android Demo](#6-android-demo)

#### [6.1 编译与构建](#61-编译与构建)

*用法：*

```
# 回到 rknn_model_zoo 根目录
cd ../../
export ANDROID_NDK_PATH=<android_ndk_path>

./build-android.sh -t <TARGET_PLATFORM> -a <ARCH> -d mms_tts

# 例如 
./build-android.sh -t rk3588 -a arm64-v8a -d mms_tts
```

*说明：*

* `<android_ndk_path>`：指定 Android NDK 路径。
* `<TARGET_PLATFORM>`：指定 NPU 平台名。支持平台见 [这里](#2-%E5%BD%93%E5%89%8D%E6%94%AF%E6%8C%81%E5%B9%B3%E5%8F%B0)。
* `<ARCH>`：指定设备系统架构。可通过以下命令查询设备架构：

```
# 查询架构。Android 设备日志中应显示 ['arm64-v8a' 或 'armeabi-v7a']。
adb shell cat /proc/version
```

#### [6.2 推送 Demo 文件到设备](#62-推送-demo-文件到设备)

设备通过 USB 连接后，将 Demo 文件推送到设备：

```
adb root
adb remount
adb push install/<TARGET_PLATFORM>_android_<ARCH>/rknn_mms_tts_demo/ /data/
```

#### [6.3 运行 Demo](#63-运行-demo)

```
adb shell
cd /data/rknn_mms_tts_demo

export LD_LIBRARY_PATH=./lib
./rknn_mms_tts_demo model/mms_tts_eng_encoder_200.rknn model/mms_tts_eng_decoder_200.rknn '"Mister quilter is the apostle of the middle classes and we are glad to welcome his gospel."'

# 如需拉长语音时长（例如修正 2 倍速听感），可增加 speaking_rate 参数（越小越慢）
./rknn_mms_tts_demo model/mms_tts_eng_encoder_200.rknn model/mms_tts_eng_decoder_200.rknn '"Mister quilter is the apostle of the middle classes and we are glad to welcome his gospel."' 0.5
```

* 运行后会生成 `output.wav`。如需在主机查看结果，可执行：

```
adb pull /data/rknn_mms_tts_demo/output.wav
```

## [7. Linux Demo](#7-linux-demo)

请注意，Linux 编译工具链推荐使用 `gcc-linaro-6.3.1(aarch64)/gcc-arm-8.3(armhf)/armhf-uclibcgnueabihf(armhf for RV1106/RV1103 series)`。使用其他版本可能会遇到 C demo 编译失败的问题。详细编译说明请参考 [Compilation\_Environment\_Setup\_Guide.md](../../docs/Compilation_Environment_Setup_Guide.md)。

#### [7.1 编译与构建](#71-编译与构建)

*用法*

```
# 回到 rknn_model_zoo 根目录
cd ../../

# 若构建时找不到 GCC_COMPILER，请设置 GCC_COMPILER 路径
(optional)export GCC_COMPILER=<GCC_COMPILER_PATH>

./build-linux.sh -t <TARGET_PLATFORM> -a <ARCH> -d mms_tts

# 例如 
./build-linux.sh -t rk3588 -a aarch64 -d mms_tts
```

*说明：*

* `<GCC_COMPILER_PATH>`：指定 GCC\_COMPILER 路径。
* `<TARGET_PLATFORM>`：指定 NPU 平台名。支持平台见 [这里](#2-%E5%BD%93%E5%89%8D%E6%94%AF%E6%8C%81%E5%B9%B3%E5%8F%B0)。
* `<ARCH>`：指定设备系统架构。可通过以下命令查询设备架构：

```
# 查询架构。Linux 设备日志中应显示 ['aarch64' 或 'armhf']。
adb shell cat /proc/version
```

#### [7.2 推送 Demo 文件到设备](#72-推送-demo-文件到设备)

* 若设备通过 USB 连接，可将 Demo 文件推送到设备：

```
adb push install/<TARGET_PLATFORM>_linux_<ARCH>/rknn_mms_tts_demo/ /data/
```

* 对于其他板卡，可使用 `scp` 或其他方式将 `install/<TARGET_PLATFORM>_linux_<ARCH>/rknn_mms_tts_demo/` 下全部文件拷贝到 `data` 目录。

#### [7.3 运行 Demo](#73-运行-demo)

```
adb shell
cd /data/rknn_mms_tts_demo

export LD_LIBRARY_PATH=./lib
./rknn_mms_tts_demo model/mms_tts_eng_encoder_200.rknn model/mms_tts_eng_decoder_200.rknn '"Mister quilter is the apostle of the middle classes and we are glad to welcome his gospel."'

# 如需拉长语音时长（例如修正 2 倍速听感），可增加 speaking_rate 参数（越小越慢）
./rknn_mms_tts_demo model/mms_tts_eng_encoder_200.rknn model/mms_tts_eng_decoder_200.rknn '"Mister quilter is the apostle of the middle classes and we are glad to welcome his gospel."' 0.5
```

* 运行后会生成 `output.wav`。如需在主机查看结果，可执行：

```
adb pull /data/rknn_mms_tts_demo/output.wav
```
