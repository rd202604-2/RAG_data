# [Rockchip RKNPU / RKNN SDK V2.3.2 — 四份核心 PDF 阅读指南](#rockchip-rknpu--rknn-sdk-v232--四份核心-pdf-阅读指南)

本文说明 `rknn-toolkit2/doc/` 目录下四份中文 PDF 各自**定位、读者、建议阅读顺序**，便于与「PC 端转换」和「板端部署」工作流对齐。文内对文档目的的概括，均来自各 PDF **前言/概述**中的原文表述。

---

## [1. 四份文档一句话对照](#1-四份文档一句话对照)

| 序号 | 文件名 | 文档标题（PDF 内） | 干什么用 |
| --- | --- | --- | --- |
| **01** | `01_Rockchip_RKNPU_Quick_Start_RKNN_SDK_V2.3.2_CN.pdf` | RKNN SDK 快速上手指南 | **零基础**：从装环境到跑通 Model Zoo / Python Demo / C Demo 的**最短路径** |
| **02** | `02_Rockchip_RKNPU_User_Guide_RKNN_SDK_V2.3.2_CN.pdf` | RKNN SDK User Guide | **系统手册**：软件栈、完整开发流程、转换/评估/板端推理、进阶主题与示例 |
| **03** | `03_Rockchip_RKNPU_API_Reference_RKNN_Toolkit2_V2.3.2_CN.pdf` | RKNN-Toolkit2 API 参考手册 | **PC 端 Python**：`RKNN` 对象从配置、加载、构建、导出到连板推理/量化等 **API 速查** |
| **04** | `04_Rockchip_RKNPU_API_Reference_RKNNRT_V2.3.2_CN.pdf` | RKNN C API 参考手册 | **设备端 C**：`rknn_*` 初始化、输入输出、内存、Matmul、自定义算子等 **板端运行时 API 速查** |

---

## [2. 各文档具体在讲什么（按官方表述）](#2-各文档具体在讲什么按官方表述)

### [2.1 `01` — 快速上手指南（约 27 页）](#21-01--快速上手指南约-27-页)

**官方定位**（概述）：面向零基础用户，说明如何在计算机上使用 **RKNN-Toolkit2** 完成模型转换，并通过 **RKNPU2** 部署到 Rockchip 开发板。

**典型目录脉络**（摘自 PDF 目录）：准备开发板 → 准备开发环境（仓库、Toolkit2、编译工具、板端 RKNPU2 环境）→ **运行示例程序**（Model Zoo、Python Demo、C Demo，含可选精度评估）。

**适合**：第一次接触 RKNN、需要「跟着做一遍就亮」的工程师。

---

### [2.2 `02` — User Guide / 用户指南（约 165 页）](#22-02--user-guide--用户指南约-165-页)

**官方定位**（前言概述）：介绍 **Rockchip RKNN SDK** 的开发。

**内容广度**（摘自 PDF 目录，仅列一级结构）： RKNN 简介与开发流程 → 开发环境准备（Docker / Pip、板端 NPU、RKNN Server）→ **RKNN 使用说明**（模型转换、评估、板端 C/Python 推理、Matmul）→ **示例**（MobileNet、YOLOv5）→ **进阶**（数据排布、零拷贝、多核、动态 Shape、自定义算子、多 Batch、SRAM、剪枝、加密、稀疏化等）。

**适合**：已跑通示例后，需要**系统化理解整条链路**、查「怎么做某类任务」、对照进阶能力（零拷贝、动态输入等）时阅读。

---

### [2.3 `03` — RKNN-Toolkit2 API 参考（约 22 页）](#23-03--rknn-toolkit2-api-参考约-22-页)

**官方定位**（前言概述）：**RKNN-Toolkit2** 的 API 参考；Toolkit2 用于在 **PC 平台**上进行模型**转换、推理和性能评估**。

**典型章节**（摘自 PDF 目录）：适用芯片与依赖 → **API 详细说明**（初始化/释放、模型配置、各框架加载、构建/导出 RKNN、加载 RKNN、初始化运行时、推理、性能评估、内存、混合量化、自定义算子、生成 C++ 部署示例等）。

**适合**：在 **Ubuntu 等主机**上写 `convert.py`、调 `config`、连板评估时，**查参数名、函数行为、调用顺序**。

---

### [2.4 `04` — RKNN Runtime / C API 参考（约 52 页）](#24-04--rknn-runtime--c-api-参考约-52-页)

**官方定位**（前言概述）：**Rockchip RKNN C API** 参考手册。

**典型章节**（摘自 PDF 目录）：概述 → 硬件平台 → **RKNPU 编译说明**（头文件、Linux/Android/RK2118 等运行时库）→ **C API 说明**（数据结构、`rknn_init` / `rknn_run`、内存与零拷贝相关接口、Matmul API、自定义算子等）。

**适合**：在 **板端或交叉编译环境**里集成 `librknnrt.so`、写 C/C++ 推理、对接 DMA-BUF/物理内存、做高性能路径时**查 C 接口与标志位**。

---

## [3. 该怎么读：建议顺序与分工](#3-该怎么读建议顺序与分工)

用下面一张**角色分工**图理解四份文档在链路中的位置：

```
flowchart LR
subgraph PC["PC（Ubuntu 等）"]
T2["RKNN-Toolkit2\nPython"]
end
subgraph Board["开发板（Linux / Android）"]
RT["RKNN Runtime\nC/C++"]
end
T2 -->|"生成 .rknn"| RT
```

| 阶段 | 优先读 | 补充读 |
| --- | --- | --- |
| 入门、第一次跑通 | **01** | 卡住时再翻 **02** 对应章节 |
| 理解 SDK 全貌、部署与进阶 | **02** | 写代码时 **03** + **04** 当词典 |
| 主机上转换/量化/评估脚本 | **03** | 概念与流程以 **02** 第 3 章为准 |
| 板端 C Demo / 自研推理引擎 | **04** | 流程与优化思路见 **02**（如零拷贝、多核、动态 Shape） |

**记忆口诀**：

* **01**：能跑再说。
* **02**：为什么这样跑、还能怎么跑。
* **03**：主机 Python 里**函数怎么调**。
* **04**：板子 C 里**函数怎么调**。

## [4. `03` 与 `04` 最易混淆：务必区分运行环境](#4-03-与-04-最易混淆务必区分运行环境)

| 维度 | `03` RKNN-Toolkit2 | `04` RKNN Runtime（RKNNRT） |
| --- | --- | --- |
| **语言/环境** | 主要是 **Python**（PC 安装 `rknn_toolkit2`） | **C API**（链接 `librknnrt`） |
| **典型用途** | ONNX 等 → 构建 → 导出 `.rknn`；连板**调试/评估** | 加载 `.rknn`，在 NPU 上 **rknn\_run** |
| **查什么** | `RKNN.config` / `load_onnx` / `build` / `export_rknn` 等 | `rknn_init` / `rknn_inputs_set` / `rknn_run` / `rknn_create_mem` 等 |

同一条业务链路里：**先在 PC 用 Toolkit2 产出模型（对照 03），再在设备用 Runtime 推理（对照 04）**；**02** 把两段的配置前提、板端服务和进阶能力串在一起。

---
