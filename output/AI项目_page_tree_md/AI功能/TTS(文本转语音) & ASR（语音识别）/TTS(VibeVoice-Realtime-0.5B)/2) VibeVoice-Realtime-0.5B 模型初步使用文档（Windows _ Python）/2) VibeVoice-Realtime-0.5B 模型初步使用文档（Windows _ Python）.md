# [VibeVoice-Realtime-0.5B 使用文档（Windows / Python）](#vibevoice-realtime-05b-使用文档windows--python)

* 官方模型卡：`models/VibeVoice-Realtime-0.5B/README.md`

## [1. 模型加载示例（Python）](#1-模型加载示例python)

> 说明：`VibeVoice-Realtime-0.5B` 当前官方推荐通过 `microsoft/VibeVoice` 仓库中的推理脚本使用。 该脚本实际加载类（见官方脚本 import）：
>
> * `vibevoice.modular.modeling_vibevoice_streaming_inference.VibeVoiceStreamingForConditionalGenerationInference`
> * `vibevoice.processor.vibevoice_streaming_processor.VibeVoiceStreamingProcessor`

### [1.1 先准备官方代码（一次性）](#11-先准备官方代码一次性)

```
cd C:\Htek\ARSTTS
git clone https://github.com/microsoft/VibeVoice.git
cd .\VibeVoice
..\ .venv\Scripts\python -m pip install -e .[streamingtts]
```

如果你直接复制上面命令，请把 `..\ .venv` 中间空格删掉，正确写法是 `..\.venv`。

### [1.2 Python 中加载模型（与官方脚本一致）](#12-python-中加载模型与官方脚本一致)

```
from vibevoice.modular.modeling_vibevoice_streaming_inference import (
VibeVoiceStreamingForConditionalGenerationInference,
)
from vibevoice.processor.vibevoice_streaming_processor import VibeVoiceStreamingProcessor

model_path = r"C:\Htek\ARSTTS\models\VibeVoice-Realtime-0.5B"

processor = VibeVoiceStreamingProcessor.from_pretrained(model_path)
model = VibeVoiceStreamingForConditionalGenerationInference.from_pretrained(
model_path,
device_map="cuda", # 无 GPU 可改为 "cpu"
torch_dtype="auto",
attn_implementation="flash_attention_2", # 若报错可改 "sdpa"
)
model.eval()
print("model + processor loaded")
```

## [3. 关键调用链（基于官方推理脚本）](#3-关键调用链基于官方推理脚本)

下面是 `demo/realtime_model_inference_from_file.py` 的核心流程（来源：官方 raw 文件）：

1. 参数解析：`parse_args()`
2. 加载处理器：`VibeVoiceStreamingProcessor.from_pretrained(args.model_path)`
3. 加载模型：`VibeVoiceStreamingForConditionalGenerationInference.from_pretrained(...)`
4. 文本与提示处理：`processor.process_input_with_cached_prompt(...)`
5. 语音生成：`model.generate(...)`
6. 保存音频：`processor.save_audio(...)`

数据流示意图：

```
txt_path(输入文本)
-> processor.process_input_with_cached_prompt
-> model.generate
-> outputs.speech_outputs
-> processor.save_audio
-> *.wav(落盘)
```

## [4. CPU/GPU/CUDA 检查与 GPU 配置步骤](#4-cpugpucuda-检查与-gpu-配置步骤)

### [4.0 先选路径（新卡 / 老卡）](#40-先选路径新卡--老卡)

先执行：

```
nvidia-smi
```

* 老卡路径（本机）：`sm_61`（GTX1060），走 `float32 + eager`

可用下面命令快速查计算能力：

```
cd C:\Htek\ARSTTS
.\.venv\Scripts\python -c "import torch; print(torch.cuda.get_device_name(0) if torch.cuda.is_available() else 'no cuda'); print(torch.cuda.get_device_capability(0) if torch.cuda.is_available() else 'no cuda')"
```

### [4.1 检查显卡、驱动、CUDA Runtime（系统层）](#41-检查显卡驱动cuda-runtime系统层)

```
nvidia-smi
```

* GPU: `NVIDIA GeForce GTX 1060 6GB`

### [4.2 检查 Python 环境中的 PyTorch CUDA 能力（框架层）](#42-检查-python-环境中的-pytorch-cuda-能力框架层)

```
cd C:\Htek\ARSTTS
.\.venv\Scripts\python -c "import torch; print('torch', torch.__version__); print('cuda_compiled', torch.version.cuda); print('cuda_available', torch.cuda.is_available())"
```

如果输出类似以下内容，说明当前是 CPU 版 PyTorch，不能用 `--device cuda`：

* `torch 2.11.0+cpu`
* `cuda_compiled None`
* `cuda_available False`

### [4.3 关键兼容性结论（GTX1060 / sm\_61）](#43-关键兼容性结论gtx1060--sm-61)

你这台机器是 `GTX 1060 6GB`（计算能力 `sm_61`）。 我们已实测：`PyTorch 2.11.0+cu128` 会报 `no kernel image is available for execution on the device`，原因是该 wheel 不包含 `sm_61` 内核。

另外，当前系统仅安装了 `Python 3.14`（`py -0p` 实测）。 旧版可支持 `sm_61` 的 PyTorch CUDA wheel 一般不提供 `cp314` 包，因此需要单独准备 `Python 3.10` 虚拟环境。

### [4.4 新卡（sm >= 75）推荐方案](#44-新卡sm--75推荐方案)

> 适用：Turing/Ampere/Ada 等较新显卡，且你希望保持当前主环境（如 `.venv`）直接跑 CUDA。

```
cd C:\Htek\ARSTTS
.\.venv\Scripts\python -m pip uninstall -y torch torchvision torchaudio
.\.venv\Scripts\python -m pip install --index-url https://download.pytorch.org/whl/cu128 torch torchvision torchaudio
.\.venv\Scripts\python -c "import torch; print(torch.__version__, torch.version.cuda, torch.cuda.is_available(), torch.cuda.get_device_capability(0) if torch.cuda.is_available() else 'N/A')"
.\.venv\Scripts\python .\scripts\vibevoice_first_test.py --device cuda
```

若出现 `flash_attn` 相关报错，可先继续使用 `sdpa` 或 `eager` 路径验证功能，再考虑单独安装 `flash-attn`。

### [4.9 老显卡（sm\_61）参数策略说明](#49-老显卡sm-61参数策略说明)

为兼容 GTX1060（Pascal, sm\_61），已在 `VibeVoice/demo/realtime_model_inference_from_file.py` 中对 CUDA 分支增加了老架构判断：

* `major < 8` 时使用：
* `torch_dtype = float32`
* `attn_implementation = eager`

这样可避免 `flash_attn` 缺失与 `sdpa/cutlass` 内核不兼容引发的报错（例如 `no kernel found to launch`）。
