# [VibeVoice 目录说明（本仓库克隆）](#vibevoice-目录说明本仓库克隆)

本文档位于 **ARSTTS 仓库根目录**，与 `VibeVoice-Realtime-0.5B-使用文档.md` 等并列，便于统一查阅。

说明对象为 **`microsoft/VibeVoice` 在本机的克隆**：`ARSTTS/VibeVoice/`。 除特别标明为 `ARSTTS/...` 的路径外，文中形如 `demo/...`、`vibevoice/...` 的路径均相对于 **`VibeVoice/` 目录根**（完整路径为 `ARSTTS/VibeVoice/...`）。

说明基于当前仓库中的真实路径与源码结构；若你后续在 `VibeVoice` 内 `git pull` 更新，以官方仓库为准。

---

## [1. 这个文件夹整体是干什么的](#1-这个文件夹整体是干什么的)

`VibeVoice` 是微软开源的**语音 AI 套件**：包含 **流式实时 TTS（VibeVoice-Realtime-0.5B）**、**长音频 ASR（VibeVoice-ASR）** 等模型与演示代码。 该子目录通过 `VibeVoice/pyproject.toml` 安装为 Python 包 `vibevoice`，并可选注册 **vLLM 插件** 以加速 ASR 推理。

> 官方 `VibeVoice/README.md` 中注明：历史上曾提供 **VibeVoice-TTS** 的长篇多说话人代码，后因合规原因已从本仓库移除；文档中仍保留介绍与论文链接，但**本仓库当前不包含该 TTS 训练/推理代码**。

---

| 路径（相对 `VibeVoice/`） | 作用 |
| --- | --- |

| `.gitignore` | Git 忽略规则（含 `.venv`、`venv/` 等）。 |

## [3. `vibevoice/` — Python 核心库](#3-vibevoice--python-核心库)

对外暴露的入口见 `vibevoice/__init__.py`：主要导出 **流式推理模型**与 **Processor**（用于 Realtime TTS 管线）。

### [3.1 `vibevoice/modular/` — 模型与推理实现](#31-vibevoicemodular--模型与推理实现)

| 文件 | 作用 |
| --- | --- |
| `configuration_vibevoice.py` | 非流式/通用 VibeVoice 的 `Config`（如 ASR 等场景的配置注册）。 |
| `configuration_vibevoice_streaming.py` | **流式 TTS** 的 `VibeVoiceStreamingConfig` 等。 |
| `modeling_vibevoice.py` | 基础建模逻辑（与 Transformers 风格对齐的模块组合）。 |
| `modeling_vibevoice_streaming.py` | 流式 TTS 训练/结构侧模型定义（`VibeVoiceStreamingModel` 等）。 |
| `modeling_vibevoice_streaming_inference.py` | **流式 TTS 推理**用的 `VibeVoiceStreamingForConditionalGenerationInference`；`demo/realtime_model_inference_from_file.py` 直接引用。 |
| `modeling_vibevoice_asr.py` | **ASR** 条件生成模型 `VibeVoiceASRForConditionalGeneration`。 |
| `modular_vibevoice_tokenizer.py` | 声学/语义等 **tokenizer 模块**（连续语音 token 相关）。 |
| `modular_vibevoice_text_tokenizer.py` | 文本侧 tokenizer 封装（含 ASR 用的 `VibeVoiceASRTextTokenizerFast` 等，与 vLLM 插件注册相关）。 |
| `modular_vibevoice_diffusion_head.py` | **扩散头**：与 LLM 输出配合生成声学细节。 |
| `streamer.py` | `AudioStreamer` / `AsyncAudioStreamer`：流式音频输出；`demo/web/app.py` 的 WebSocket 服务使用。 |

### [3.2 `vibevoice/processor/` — 前后处理](#32-vibevoiceprocessor--前后处理)

| 文件 | 作用 |
| --- | --- |
| `vibevoice_streaming_processor.py` | 流式 TTS 的 `VibeVoiceStreamingProcessor`（文本/语音 token 与特征管线）。 |
| `vibevoice_tokenizer_processor.py` | 与 tokenizer 协同的处理器。 |
| `vibevoice_processor.py` | 通用处理器逻辑。 |
| `vibevoice_asr_processor.py` | ASR 输入（音频等）的 `VibeVoiceASRProcessor`。 |
| `audio_utils.py` | 音频加载、重采样等工具函数。 |
| `__init__.py` | 导出上述 Processor 符号。 |

### [3.3 `vibevoice/schedule/` — 采样/求解器](#33-vibevoiceschedule--采样求解器)

| 文件 | 作用 |
| --- | --- |
| `dpm_solver.py` | DPM-Solver 类扩散采样实现（供扩散头推理使用）。 |
| `timestep_sampler.py` | 时间步采样策略。 |

### [3.4 `vibevoice/scripts/` — 工具脚本](#34-vibevoicescripts--工具脚本)

| 文件 | 作用 |
| --- | --- |
| `convert_nnscaler_checkpoint_to_transformers.py` | 将 nnScaler 等格式检查点转换为 Transformers 可用格式（迁移/对齐权重用）。 |

### [3.5 `vibevoice/configs/`](#35-vibevoiceconfigs)

| 文件 | 作用 |
| --- | --- |
| `qwen2.5_1.5b_64k.json`、`qwen2.5_7b_32k.json` | 与骨干（如 Qwen2.5）相关的配置片段，供模型组装或实验参考。 |

## [4. `demo/` — 可运行示例](#4-demo--可运行示例)

| 路径 | 作用 |
| --- | --- |
| `realtime_model_inference_from_file.py` | **从文本文件做 Realtime TTS**，加载本地或 HF 模型路径，扫描 `demo/voices/streaming_model/` 下的音色 `.pt`，输出 wav。`ARSTTS/scripts/vibevoice_first_test.py` 即调用此脚本。 |
| `vibevoice_realtime_demo.py` | 启动 **uvicorn**，加载 `web.app`，提供浏览器端流式 TTS 演示（端口等可通过参数指定）。 |
| `vibevoice_realtime_colab.ipynb` | Colab 上的流式 TTS 教程笔记本。 |
| `vibevoice_asr_inference_from_file.py` | **ASR 批量推理**示例：从文件读音频，走 `VibeVoiceASRForConditionalGeneration` + `VibeVoiceASRProcessor`。 |
| `vibevoice_asr_gradio_demo.py` | ASR 的 **Gradio** 界面演示脚本。 |
| `text_examples/1p_vibevoice.txt`、`1p_abs.txt` | TTS 示例剧本文本。 |
| `download_experimental_voices.sh` | 下载实验音色的 shell 脚本（需自行赋予执行权限并在类 Unix 环境运行）。 |
| `web/app.py` | **FastAPI + WebSocket**：流式推音频；静态页由 `web/index.html` 提供。 |
| `web/index.html` | 前端页面，与 `app.py` 配合。 |

> **音色目录**：推理脚本默认在 `demo/voices/streaming_model/` 下查找 `.pt` 音色。若该目录未随克隆出现，需按官方文档下载或运行 `download_experimental_voices.sh`。

## [8. `vibevoice.egg-info/`（可忽略）](#8-vibevoiceegg-info可忽略)

在 `VibeVoice` 目录下执行 `pip install -e .` 后由 setuptools 生成，记录包名、依赖、入口点等；**不要手改**，一般也不提交到版本库。

## [9. 在 `VibeVoice` 里「能做什么」（能力清单）](#9-在-vibevoice-里能做什么能力清单)

| 能力 | 典型入口（由 ARSTTS 根看完整路径） |
| --- | --- |
| **本地/命令行 Realtime TTS（0.5B）** | `VibeVoice/demo/realtime_model_inference_from_file.py` + `ARSTTS/models/VibeVoice-Realtime-0.5B` |
| **浏览器/WebSocket 流式 TTS** | `VibeVoice/demo/vibevoice_realtime_demo.py` → `VibeVoice/demo/web/app.py` |
| **ASR 文件批量推理** | `VibeVoice/demo/vibevoice_asr_inference_from_file.py` |
| **ASR Gradio 试玩** | `VibeVoice/demo/vibevoice_asr_gradio_demo.py` |
| **ASR LoRA 微调** | `VibeVoice/finetuning-asr/lora_finetune.py`（见该目录 `README.md`） |
| **vLLM 加速 ASR** | 安装 vLLM 后利用 `VibeVoice/pyproject.toml` 中的插件入口 + `VibeVoice/docs/vibevoice-vllm-asr.md` |
| **以库方式引用流式模型** | `from vibevoice import VibeVoiceStreamingForConditionalGenerationInference, VibeVoiceStreamingProcessor`（见 `VibeVoice/vibevoice/__init__.py`） |

## [10. 与 ARSTTS 项目其它目录的关系](#10-与-arstts-项目其它目录的关系)

* 权重常放在 `ARSTTS/models/VibeVoice-Realtime-0.5B`，通过 `--model_path` 传给上述 demo。
* `ARSTTS/scripts/vibevoice_first_test.py` **不修改** `VibeVoice` 内部逻辑，仅封装调用 `VibeVoice/demo/realtime_model_inference_from_file.py`。
